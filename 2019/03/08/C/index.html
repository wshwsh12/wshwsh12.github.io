1.Const

1.1 作用

1.修饰变量，说明变量不可修改。

2.修饰指针，分为指向常量的指针和指针常量。

3.常量引用。经常用于形参类型，避免内存拷贝，也避免函数对值修改。

4.修饰类的成员函数，说明该成员函数不能修改成员变量。

1.2 使用

1.记住*&向后结合自己推倒即可。

2.在类中修饰成员函数，如：int getValue() const;

2.static用法

1.修饰普通变量，修改变量的存储区域和声明周期，使其存储在静态区。

2.修饰普通函数。表明函数作用范围，仅为本文件内才能使用。多人开发项目时，可防止函数才重名。

3.修饰成员变量。不需要实例化就可以访问变量。

4.修饰成员函数。不需要实例化就可以访问函数。不能访问非静态成员。

3.this指针

1.this指针隐含于每个非静态成员函数的特殊指针，指向正常操作的对象。

2.调用成员函数时，先给this赋值，调用成员函数。访问数据成员时，隐含的使用this指针。

3.this的类型为 ClassName *const this 为常量指针，不能给this赋值。由const修饰的成员函数中，this的类型为 const ClassName* const this 为指向常量的常量指针，不能对数据成员进行修改。

4.this为右值，不能取地址。

4.inline内联函数

特点

1.宏+类型检查，具有函数特性。

2.不能包含递归循环等复杂操作。

3.类中除了虚函数，其他函数都会自动当成内联函数。

编译器处理

1.复制到调用点

2.分配内存空间

3.将输入和返回值映射到局部变量空间。

4.如果有多个返回点，将其转换到末尾。

优缺点

优点

1.快速，省去压栈出栈等操作。

2.会有安全检查和自动类型转换。（和普通函数相同）

3.可以作为普通函数调试。

缺点

1.代码膨胀。

2.inline函数的改变需要重新编译。non-inline可以链接。

3.是否内联，程序员不可控。

5.assert()

1.断言，是宏定义，不是函数。

2.当条件为假时，终止程序。

3.通过 #define NDEBUG 取消，头文件为 assert.h cassert。

6.pragma pack(n)

设置对其方式为n字节。

7.位域

Type var:num;

1.相同类型，能合并会合并。

2.位域作用于字节，num不能大于8。

3.num为0时跳到下个字节,变量名为空可以占位。

8.volatile

volatile int i = 10;

修饰变量，让编译器对该变量不进行优化。
